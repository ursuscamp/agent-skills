# Bug Report Format

Use this structure for `bug-report.md`:

1. `# [Bug ID]: [Brief description]`
2. `## Summary`
3. `## Severity: [Critical|High|Medium|Low]`
4. `## Environment`
5. `## Reproduction Steps`
6. `## Expected Behavior`
7. `## Actual Behavior`
8. `## Impact`
9. `## Root Cause`
10. `## Solution Approach`
11. `## Code Changes`
12. `## Edge Cases`

## Rules

- Reproduction steps must be concrete and repeatable.
- Include exact error text, values, and file locations when known.
- Root cause should explain why the bug happens, not just where.
- Solution approach should name the chosen fix and rejected alternatives.
- Code changes should identify the files and the nature of each change.
- Edge cases should cover nearby regressions and unusual inputs.
