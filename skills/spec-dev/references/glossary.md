# Glossary Workflow

Maintain project terminology in `[base-specs-dir]/glossary.md`.

## Entry format

```markdown
## [Term Name]

**Definition:** Clear, concise definition

**Context:** Where this term applies in the project

**Example:** Concrete example from the codebase, API, or workflow

**Related Terms:** Comma-separated related terms
```

## Rules

- Check for an existing entry before creating a new one.
- Keep entries alphabetized.
- Prefer concrete project examples over generic examples.
- Keep related-term references bidirectional when practical.
- Update entries when scope changes instead of duplicating them.
